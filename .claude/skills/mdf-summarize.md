# mdf-summarize

## 触发条件

用户输入 `/mdf-summarize` 或手动调用此 skill。

## 执行步骤

1. **读取变更记录**：读取当前项目 `docs/mdf/changes.md` 文件的全部内容

2. **分析内容**：
   - 阅读每条记录，分析其主题和类别
   - 查看 `docs/mdf/` 目录下已有哪些分类文件
   - 决定每条记录应该归入哪个文件（追加或新建）

3. **执行分类写入**：
   - 对于已有相关分类文件的记录，追加内容到对应文件
   - 对于新主题，创建新的分类文件
   - 保持内容有序，避免重复

4. **清空变更记录**：
   - 写入完成后，清空 `docs/mdf/changes.md` 内容（保留文件头）
   - 同时重置 `.count` 文件为 0

5. **向用户汇报**：
   - 告知用户生成了哪些分类文件
   - 汇总了多少条记录

## 分类建议

- **architecture.md**：架构设计、重构、模块划分
- **decisions.md**：技术决策、方案选择、权衡
- **preferences.md**：用户偏好、代码风格、规范
- **progress.md**：项目进展、里程碑、完成的功能
- 其他可创建合适的分类文件名
